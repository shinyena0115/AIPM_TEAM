**개발 기간**: 2024-2025
**기술 스택**: Vue.js 3, Node.js, Express, MySQL, OpenAI GPT-4o

## 담당 기능 개요

- Employee(사원) 사용자 전체 기능 구현
- 핵심 기능: AI 기반 업무 우선순위 시스템, 연차 게시판, 개인 업무 관리
- OpenAI GPT-4o 활용한 자동화에 중점

## 1. AI 기반 업무 우선순위 시스템

### 시스템 개요
- 문제: "여러 업무 중 어떤 것부터 처리해야 할까?"
- 해결: 점수 기반 자동 계산 (0.1초 이내 상위 3개 추천) + GPT-4o 심층 분석
- 주요 파일: `front/src/views/Employee/Tasks.vue`, `backend/routes/api/apiTask.js`

### 1-1. AI 문서 분석 기능
- **기능**: 회의록 이미지, 이메일 스크린샷, PDF 업로드 시 GPT-4o Vision이 자동 분석
- **처리 과정**:
  - 이미지 → Base64 인코딩 → GPT-4o Vision API 전달
  - PDF → pdf-parse로 텍스트 추출 → GPT-4o 분석
  - AI가 제목, 중요도, 소요시간, 업무 유형, 난이도 자동 추출
- **특징**: 여러 이미지를 하나의 문서로 병합 분석 가능
- **구현 위치**: `apiTask.js` 240-377줄, `Tasks.vue` 693-743줄

### 1-2. AI 텍스트 분석 기능
- **기능**: 이메일 본문, Slack/Teams 메시지 등 텍스트 직접 입력 분석
- **처리 속도**: 문서 분석보다 1-2초 빠름 (파일 변환 과정 없음)
- **구현 위치**: `Tasks.vue` 745-779줄

### 1-3. 간단 AI 분석 기능
- **기능**: 제목과 마감일만으로 나머지 정보 AI가 추론
- **처리 속도**: 3초 이내 완료
- **추론 기준**: 프롬프트에 소요시간 가이드라인 포함 (간단한 수정 30-60분, 일반 개발 120-240분, 복잡한 기능 360-480분)
- **구현 위치**: `Tasks.vue` 531-595줄, `apiTask.js` 380-431줄

### 1-4. 자동 우선순위 계산 알고리즘
- **계산 방식**:
  - 중요도: 최대 100점 (높음 100, 중간 50, 낮음 10)
  - 긴급도: 최대 200점 (마감 지남 200, 24시간 이내 80, 3일 이내 40, 그 이상 10)
  - 난이도: 최대 15점 (어려움 15, 보통 10, 쉬움 5)
- **총점**: 최대 315점
- **처리 속도**: 0.1초 이내 (서버 호출 없이 프론트엔드에서 직접 계산)
- **시각화**: 상위 3개를 색상/크기로 구분 (1순위: 굵은 초록 테두리 4px + 배지 36px)
- **구현 위치**: `Tasks.vue` 473-496줄 (계산), 417-425줄 (상위 3개 추출)

### 1-5. GPT-4o 심층 분석 기능
- **기능**: 모든 미완료 업무를 GPT-4o가 종합 분석하여 추천
- **분석 기준 5가지**:
  1. 긴급도와 마감일 (지연 업무 최우선)
  2. 중요도 (비즈니스 임팩트, 팀원 블로킹 여부)
  3. 업무 유형별 특성 (회의: CEO 참석 시 최우선, 개발: 블로킹 여부)
  4. 난이도와 소요시간 (어려운 업무는 오전 배치, 짧은 업무는 틈새시간)
  5. 업무 간 의존성 (API 개발 → 프론트 연동 순서)
- **결과 형식**: 각 업무별 선정 이유 + 권장 처리 시간 + 시간대별 일정 제안
- **처리 속도**: 3-5초
- **구현 위치**: `apiTask.js` 433-591줄, `Tasks.vue` 596-612줄

### 1-6. 타임라인 시각화 기능
- **분류 기준**:
  - 🔴 급함: 24시간 이내 마감 (빨간색 배경)
  - 🟡 보통: 1-3일 이내 마감 (노란색 배경)
  - 🟢 여유: 3일 이상 (초록색 배경)
- **D-day 표시**: "30분 남음", "5시간 남음", "D-3" / "30분 지남", "2일 지남"
- **구현 위치**: `Tasks.vue` 282-314줄 (템플릿), 646-671줄 (D-day 계산)

### 구현 성과
- 업무 등록 시간 70% 단축
- 우선순위 결정 시간 사실상 제로
- 마감 놓치는 업무 감소
- 팀 블로킹 업무 우선 처리 문화 형성

## 2. 연차 게시판 (업무 인수인계 시스템)

### 시스템 개요
- 문제: 연차 중 발생한 업무를 복귀 후 파악하기 어려움
- 해결: 연차자에게 업무 전달 메시지 시스템 구축
- 주요 파일: `front/src/views/Employee/VacationNotice.vue`, `backend/routes/employee/VacationNotice.js`

### 2-1. 업무 전달 기능
- **과정**: 연차 중인 동료 선택 → 업무 내용 입력 → 전달
- **자동화**: 날짜는 오늘로 자동 설정 (사용자 피드백 반영)
- **필터링**: 연차 중인 동료만 자동 표시 (전체 동료 보기 기능 제거로 UX 개선)
- **DB 저장**: `ReplacementEntry` 테이블 (연차자 ID, 작성자 ID, 날짜, 내용)
- **구현 위치**: `VacationNotice.vue` 336-367줄, `VacationNotice.js` 46-73줄

### 2-2. 받은 업무 확인 기능
- **자동 로드**: 페이지 접속 시 내가 받은 모든 메시지 자동 조회
- **정렬**: 최신순
- **표시 정보**: 작성자 이름, 작성 날짜, 업무 내용
- **구현 위치**: `VacationNotice.vue` 370-385줄, `VacationNotice.js` 86-104줄

### 2-3. Tasks 시스템과의 통합
- **기능**: "업무로 추가" 버튼 → 메시지 내용이 업무 제목으로 설정 → AI 자동 분석
- **효과**: 받은 메시지를 바로 업무로 전환 → 자동 우선순위 계산

### 2-4. 작성한 업무 조회 기능
- **기능**: 내가 다른 동료에게 전달한 업무 목록 확인
- **활용**: 팔로우업 추적
- **구현 위치**: `VacationNotice.js` 106-133줄

### 구현 성과
- 연차 복귀 시 업무 누락 제로
- 업무 전달 시간 80% 단축 (날짜 자동 설정)
- 실시간 협업 강화
- Tasks 시스템과 완전 통합
- 연차 중인 동료만 표시하여 사용자 경험 개선

## 3. 개인 업무 관리 시스템 (Tasks)

### 시스템 개요
- AI 우선순위 시스템과 연차 게시판의 기반
- 기본 CRUD 기능 제공
- 주요 파일: `front/src/views/Employee/Tasks.vue`, `backend/routes/employee/tasks.js`

### 3-1. 업무 생성 기능
- **4가지 방법**:
  1. AI 문서 분석 (이미지/PDF 업로드)
  2. AI 텍스트 분석 (텍스트 입력)
  3. 간단 AI 분석 (제목+마감일)
  4. 수동 입력 (고급 옵션)
- **API**: `POST /api/tasks`
- **DB**: `tasks` 테이블에 저장

### 3-2. 업무 조회 기능
- **자동 로드**: 페이지 접속 시 내 업무 목록 조회
- **자동 분리**: 진행 중 업무 / 완료된 업무
- **API**: `GET /api/tasks`
- **구현**: Vue computed 속성으로 실시간 필터링

### 3-3. 업무 완료 처리
- **API**: `PATCH /api/tasks/:id/complete`
- **업데이트**: `completed` → true, `completedAt` → 현재 시각
- **효과**: 자동으로 완료 섹션 이동 + AI 추천 목록 재계산

### 3-4. 업무 삭제 기능
- **보안**: 확인 대화상자 + 본인 업무만 삭제 가능
- **API**: `DELETE /api/tasks/:id`
- **효과**: 배열에서 제거 + AI 추천 목록 재계산

### 구현 성과
- 체계적인 CRUD 구조
- 완료율 시각화 자동화
- AI 시스템과 완전 통합

## 기술 스택 및 구현 상세

### 프론트엔드
- **프레임워크**: Vue.js 3 Options API
- **HTTP 통신**: Axios (withCredentials: true)
- **반응성**: Computed 속성 활용 (필터링, 우선순위 계산, 타임라인 분류)
- **초기 로딩**: mounted 훅 + async/await

### 백엔드
- **프레임워크**: Node.js + Express.js
- **API 설계**: RESTful (리소스별 라우터 분리, /api prefix)
- **ORM**: Sequelize (JOIN 쿼리는 include 옵션 활용)
- **인증**: Express-session (세션 기반 인증)

### AI
- **모델**: OpenAI GPT-4o (텍스트), GPT-4o Vision (이미지)
- **설정**: temperature 0.7 (창의성과 일관성 균형)
- **PDF 처리**: pdf-parse 1.1.1
- **프롬프트**: 구조화된 JSON 반환 + 상세한 분석 기준 + 예시 포함

### 데이터베이스
- **DBMS**: MySQL
- **주요 테이블**:
  - Tasks: 업무 정보 (user_id, completed, completedAt 등)
  - ReplacementEntry: 연차 게시판 (leaver_id, author_id, 내용)
- **무결성**: 외래키 제약조건 + 사용자 삭제 금지
